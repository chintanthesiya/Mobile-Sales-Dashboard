# Mobile-Sales-Dashboard

1. Project Title
Motorola Sales Dashboard – All Month Analytics

2. Short Description / Purpose
This interactive dashboard visualizes Motorola’s mobile sales performance across India for All Months. It helps stakeholders analyze sales trends, customer behavior and payment preferences to drive data-informed decisions.

3. Tech Stack
- Frontend: Power BI 
- Backend: CSV/Excel data source or SQL database.
- Data Source:Kaggle practice set.
- Visualization: Bar charts, pie charts, line graphs, maps and using sclicer.
- Deployment: GitHub Pages and Linkdin Profile.

4. Features & Highlights
- 📍 City-wise Sales Map: Visualizes total sales across 15+ cities.
- 📅 Daily Quantity Graph: Tracks units sold per day in certain month.
- ⭐ Customer Ratings Breakdown: Shows rating distribution and satisfaction metrics.
- 💳 Payment Method Analysis: Compares UPI, credit/debit cards, and cash usage.
- 📱 Brand & Model Performance: Highlights top-selling brands and models.
- 📈 Month Sales Trends: Reveals which days drive the most revenue.
- 📐 Header KPIs: Total sales, quantity, transactions, and average sale value.

 5. Business Problem
Motorola needed a clear view of its which month sales performance across India is lower:
- Identify top-performing cities and models.
- Understand customer satisfaction levels.
- Optimize payment method offerings.
- Detect low-performing days or regions.

6. Solution
This dashboard provides:
- All month clear view dashboard to provide thair sales,quentity,transaction detail in certain month.
- Actionable insights for marketing, logistics, and customer service teams.
- A foundation for monthly comparisons and forecasting.
- Easy-to-read charts for executive reporting.
